# Roadmap

Фичи режем вертикально: lib → actions → UI, коммит после каждого слоя.

## М1 — Основной поток ✅

- [x] **Add transaction** — форма (расход/доход), `useActionState`, `useOptimistic`
- [x] **Transaction list** — список с оптимистичными апдейтами
- [x] **Stats / Dashboard** — баланс, доходы/расходы по валютам
- [x] **Редактирование / удаление** транзакции
- [x] **Выбор валюты** — RSD (default), RUB, USD, EUR

## М2 — Многопользовательность ✅

- [x] **Auth** — Google OAuth + credentials, защита роутов
- [x] **Wallet init** — автосоздание при первом входе
- [x] **Управление кошельками** — создание, переименование, удаление, переключение (sidebar + /wallets)
- [x] **Шаринг кошелька** — инвайт-ссылка, страница принятия, управление участниками, только owner может управлять
- [x] **Гостевой режим** — вход без регистрации, httpOnly cookie для восстановления сессии, шаринг скрыт для гостей

## М3 — Расширение

- [ ] **Telegram bot** — вебхук → вызов lib/ → ответ
- [ ] **Офлайн-режим** — IDB-зеркало, queue-first запись, /api/sync, SW-кеш страниц. Подробный план: [docs/offline.md](offline.md)

## Отложено

- [ ] **Фильтры** — по категории, типу, дате
- [ ] **Категории** — управление списком категорий
- [ ] **Гостевой режим: миграция данных** — при регистрации/входе гостя переносить его транзакции на новый аккаунт
- [ ] **Гостевой режим: очистка** — cron-job для удаления старых гостевых аккаунтов из БД
- [ ] **Гостевой режим: кросс-девайс** — невозможно без регистрации; рассмотреть QR-код или passphrase как способ переноса на другое устройство
