# Roadmap

Фичи режем вертикально: lib → actions → UI, коммит после каждого слоя.

## М1 — Основной поток ✅

- [x] **Add transaction** — форма (расход/доход), `useActionState`, `useOptimistic`
- [x] **Transaction list** — список с оптимистичными апдейтами
- [x] **Stats / Dashboard** — баланс, доходы/расходы по валютам
- [x] **Редактирование / удаление** транзакции
- [x] **Выбор валюты** — RSD (default), RUB, USD, EUR

## М2 — Многопользовательность ✅

- [x] **Auth** — Google OAuth + credentials, защита роутов
- [x] **Wallet init** — автосоздание при первом входе
- [x] **Управление кошельками** — создание, переименование, удаление, переключение (sidebar + /wallets)
- [x] **Шаринг кошелька** — инвайт-ссылка, страница принятия, управление участниками, только owner может управлять
- [x] **Гостевой режим** — вход без регистрации, httpOnly cookie для восстановления сессии, шаринг скрыт для гостей

## М3 — Расширение

- [ ] **Telegram bot** — вебхук → вызов lib/ → ответ
- [ ] **Офлайн-режим** — IDB-зеркало, queue-first запись, /api/sync, SW-кеш страниц. Подробный план: [docs/offline.md](offline.md)
- [x] **Persistence предпочтений** — валюта/тип транзакции и валюта отображения в cookie (сервер засевает при SSR → без вспышки), `useCookieState` поверх контекста; тема осталась на `useLocalStorage`
- [x] **Тёмная тема** — light/dark/system, inline-скрипт против мерцания, `useTheme` поверх `useLocalStorage`, переключатель в сайдбаре и на странице кошельков

## М4 — Фильтры & Аналитика

- [x] **Категории** — список-константа в `lib/` + enum в БД (единый `CATEGORY_KEYS`); авто-присвоение по ключевым словам (`suggestCategory`), select в форме с ручным override; заметка = `description`, категория = ось аналитики
- [x] **Фильтры** — по категории, типу, дате; client-side над уже загруженным массивом (`filterTransactions`)
- [x] **Выборки по времени** — неделя / месяц / квартал / произвольный диапазон (`getPeriodRange`)
- [x] **Графики** — pie по категориям + динамика по времени (line) на recharts; таб «Аналитика», клик по сектору фильтрует категорию
- [ ] **Импорт / экспорт** — выгрузка и загрузка транзакций через JSON и CSV. Логика сериализации/парсинга в `lib/` (чистые функции, переиспользуются ботом), action — тонкая обёртка с валидацией Zod. Экспорт учитывает активные фильтры; импорт — превью + дедуп перед записью

## М5 — Мульти-валюта

- [ ] **Кошельки в разных валютах** — у каждого кошелька своя валюта
- [ ] **Единый баланс** — суммирование всех кошельков в одной базовой валюте (курс ЦБ или ручной)
- [x] **Отображение в одной валюте** — пересчёт через exchangerate-api.com (24h кэш), пилюли-переключатели, выбор в cookie
- [ ] **Перевод между кошельками** — списание с одного, зачисление на другой (с учётом курса)
- [ ] **Заморозка операций** — статус «pending»: списание прошло, пополнение ещё нет

## М6 — AI & Автоматизация

- [ ] **Автоклассификация по категориям** — базовый вариант уже есть: keyword-правила (`suggestCategory`, 0 токенов) проставляют категорию при пустом поле. Этот пункт — про локальную ML-модель на будущее, если правил перестанет хватать
- [x] **Распознавание чека по фото** — Claude Vision (Sonnet 4.6) парсит чек, автозаполняет сумму, валюту, категорию и дату; HEIC/JPEG/PNG/WebP; лимит сканирований в день per user

## Отложено

- [ ] **Real-time sync** — WebSockets/SSE для одновременного редактирования несколькими пользователями (когда двое сидят в одном кошельке). `updateTag` обновляет UI только после действия — это не покрывает изменения от другого юзера без перезагрузки.

- [ ] **Метки/теги** — произвольные теги на транзакцию, фильтрация по ним
- [ ] **Получатель** — поле «кому/откуда» на транзакцию
- [ ] **Комментарий** — произвольный текст к транзакции
- [ ] **Повторяющиеся операции** — шаблон с расписанием (ежемесячная аренда и т.п.)
- [ ] **Гостевой режим: миграция данных** — при регистрации/входе гостя переносить его транзакции на новый аккаунт
- [ ] **Гостевой режим: очистка** — cron-job для удаления старых гостевых аккаунтов из БД
- [ ] **Гостевой режим: кросс-девайс** — невозможно без регистрации; рассмотреть QR-код или passphrase как способ переноса на другое устройство
