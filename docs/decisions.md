# Лог решений

## Платформа: PWA на Next, а не Telegram Mini App / нативка

- Next максимально раскрывает React 19 (Server Actions, RSC, useOptimistic).
- PWA даёт «мобилко-подобный» опыт без App Store и React Native.
- Telegram отложен: добавим позже как ещё один клиент поверх lib/.

## Бот: чат-бот, не Mini App (когда дойдём)

- Чат-бот дешевле допилить: нет UI, только парсинг → вызов lib → ответ.
- Удобнее в быту для ввода («кофе 350» на ходу).
- Условие безболезненной миграции: вся логика в lib/, userId с первого дня.

## ORM: Drizzle

- TS-first, изучаем новое после Prisma.

## БД: PostgreSQL → Neon

- Serverless-драйвер нативно дружит с Vercel Functions (нет постоянного соединения).
- Интеграция с Vercel в одну кнопку из dashboard.
- Бесплатного tier хватит для одного юзера.

## Auth: Auth.js v5

- Clerk отпал: хотим разобраться в auth самостоятельно.
- Auth.js v5 — опенсорс, Drizzle-адаптер есть, хорошая интеграция с Next.js App Router.
- userId всегда берётся из сессии Auth.js, передаётся явно в lib/.
- Планируется шаринг между юзерами → нужна authorization поверх auth (отдельная задача).

## Валидация: Zod

- Используется в Server Actions для валидации входных данных.
- В lib/ не используется — туда приходят уже валидные данные.

## Форматирование: Prettier

- create-next-app не включает Prettier, добавляем вручную.

## Тесты: Vitest

- Тестируем только `lib/` — чистые функции, легко покрываются unit-тестами.
- Компоненты и Server Actions не тестируем (pet-проект, не продакшн).

## Package manager: pnpm

- Быстрее npm, строгий резолвер зависимостей, рекомендован для Next/Vercel.

## Стили: Tailwind + shadcn/ui

- Mantine отпал: все компоненты клиентские, мешает изучению RSC.
- shadcn/ui — компоненты копируются в проект (твой код, не node_modules), RSC-совместимо.
- Tailwind: ИИ хорошо генерирует, стандарт в Next.js экосистеме.
